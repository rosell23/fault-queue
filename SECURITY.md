# Security Policy

The following sections will tell you how to report a vulnerability on `fq`.

## Supported Versions



| Version      | Supported          |
|:-------------| ------------------:|
| `<LTS>`      | :white_check_mark: |
| `dev`        | :x:                |
| `release`    | :white_check_mark: |
| `prerelease` | :x:                |
| `bugfix`     | :x:                |

## Reporting a Vulnerability

If the vulnerability is not critical, you **_may_** report it through the `Issues` tab,
or through [this link](https://github.com/rosell23/fault-queue/issues). You can also report
through [email](mailto:mario@mariorosell.es). Please, be clear about the issue and how to reproduce it.
